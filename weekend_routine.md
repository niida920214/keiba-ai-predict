# 週末のデータ更新・運用ルーティン

AI競馬予測システムを最新の状態に保ち、精度を維持するための毎週末の運用手順書です。

## 1. 概要
中央競馬の開催（原則土日）が終了した後、最新のレース結果を取り込み、予測データとモデルを更新します。
システムの基幹スクリプトである `main.py` は**差分更新**（すでに取得済みのデータはスキップし、新規データのみ取得する）に対応設計されているため、毎週末の実行により短時間で最新状態に追従できます。

> [!IMPORTANT]
> `update_data.py` は raw データの取得・結合のみを行う簡易スクリプトです。予測に必要な前処理（`data_c.pickle` の生成など）が行われないため、週末の更新には必ず **`main.py`** を使用してください。

## 2. 更新手順（毎週日曜の全レース終了後）

### Step 1: 取得期間の確認・設定
`main.py` の L35 付近にある取得期間を確認します。
基本的には、現在の日付（月）が含まれるように設定されていれば問題ありません。設定期間内の開催日から、まだ取得していないレースIDを自動で特定して差分取得します。

```python
# main.py
# ★ ここで取得期間を指定する（yyyy-mm 形式）
FROM_DATE = "2024-07"
TO_DATE = "2025-12"  # 現在の月が含まれる十分先の日付にしておく
```

### Step 2: データ更新・前処理の実行
ターミナルを開き、以下のコマンドを実行します。

```bash
python main.py
```

**実行される処理:**
1. **フェーズ0**: 指定期間内のレースIDをネットケイバカレンダから自動抽出します。
2. **フェーズ1**: 既存のローカルデータ（`data/raw/`）と照合し、存在しない**新着レースのみ**をスクレイピングします。
   - レース結果、馬の過去成績、血統、払い戻しデータを取得し既存ファイルに追記します。
3. **フェーズ2**: 結合した最新の全データを用いて一から前処理パイプラインを実行し、機械学習用の特徴量データ（`data_c.pickle`）を再生成します。

> [!NOTE]
> 週末の差分取得であれば、スクレイピング対象は数十レース単位となるため、数分〜十数分程度で完了します。

## 3. モデルの再学習（適宜）

データが更新されても、予測モデル（`lgbm_model.pickle`）は自動では更新されません。
定期的に最新のトレンドをモデルに学習させる必要があります。

### 推奨頻度
- **月1回程度**、またはG1シーズンなどの区切りでの再学習を推奨します。毎週末行う必要は必ずしもありませんが、実行しても問題ありません。

### 実行手順
最新の `data_c.pickle` が生成されている状態で、以下のコマンドを実行します。

```bash
python train_model.py
```

Optunaによるハイパーパラメータチューニング（200 Trials）と、Platt Scalingによる確率補正（Calibration）が行われるため、学習には環境により数十分〜数時間かかります。
完了すると `model/lgbm_model.pickle` が、より精度の高い予測確率を出力できるモデルとして上書き更新されます。

## 4. シミュレーション評価（学習後・任意）

モデルを再学習した場合や、新しいデータでの回収率推移を確認したい場合はシミュレーションを実行します。

```bash
python simulate.py
```

`results/` ディレクトリに以下の評価結果が出力されます。
- `tansho_return_rate.png`（単勝回収率グラフ）
- `return_rate_vs_nbets.png`（賭け枚数と回収率の関係）
- `sharpe_ratio.png`（シャープレシオ）
- `simulation_summary.csv`（数値サマリ）
- `umaren_box_return_rate.png`（馬連BOX回収率グラフ）

## 5. 予測機能の実行（次週の予想）

学習済みモデルを用いて、これから開催される特定のレースの勝率（確率）と推定オッズを予測します。
`predict.py` は、ターゲットとなる**12桁の「レースID」**を引数に取ります。

### レースIDの確認方法
netkeibaの出馬表ページ（対象レース）にアクセスし、URLの末尾にある `race_id=` 以降の12桁の数字を確認してください。
> 例: `https://race.netkeiba.com/race/shutuba.html?race_id=202605010811` の場合、レースIDは `202605010811` です。

### 実行手順
ターミナルを開き、コマンドラインから以下の形式で実行します。

```bash
# 基本形: python predict.py [レースID]
python predict.py 202605010811
```

> [!TIP]
> 開催日を明示的に指定する場合は、`--date` オプションを使用します（指定がない場合はデフォルト値が使用されます）。
> ```bash
> python predict.py 202605010811 --date 2026/02/22
> ```

### 予測結果の出力
実行が完了すると、コンソール上に各馬の勝率予測や推定オッズなどが表形式で表示されます。また、同階層に `prediction_[レースID].csv` というファイル名で予測結果が自動保存されます。

#### 予測結果（各指標）の見方
出力される各種指標の学術的・実践的な意味は以下の通りです：

- **`Prob` (AI予測勝率)**: AIが算出した各馬が「1着になる」確率（全馬合計で1になるように正規化済）。
- **`Top3` (AI予測複勝率)**: 各馬が「3着以内に入る」確率。Harvilleの公式（単勝確率から2・3着確率を導出する数理モデル）を用いて算出されています。
- **`ImpliedOdds` (AI適正オッズ)**: AIの予測勝率から逆算された「本来あるべき単勝オッズ」（$1 \div \text{Prob}$）。
- **`RealOdds` (実際の単勝オッズ)**: JRA公式（またはnetkeiba）からリアルタイムに取得した現在の単勝オッズ。
- **`TrueProb` (市場予測勝率)**: 実際のオッズから、JRAの控除率（オーバーラウンド）による歪みを排除して逆算した「一般の馬券購入者が予測している真の勝率」。
- **`TrueEV` (真の期待値)**: 期待値（Expected Value）。「AI予測勝率(`Prob`) $\times$ 実際のオッズ(`RealOdds`)」で計算され、**1.0以上なら「賭け続ければ理論上プラスになる」**ことを示します。
- **`Edge` (エッジ/優位性)**: 「AI予測勝率(`Prob`) $-$ 市場予測勝率(`TrueProb`)」。この値がプラスに大きいほど、**市場（一般のファン）よりもAIがその馬を高く評価している（過小評価されている期待値の高い穴馬）**ことを意味します。
- **`Kelly` (ケリー基準)**: 複利を最大化する最適な資金配分比率（Kelly Criterion）。「$\text{Edge} \div (\text{RealOdds} - 1)$」で算出され、この値が大きい馬ほど投資資金を厚く賭けるべきという目安になります。

> [!NOTE]
> 現在の `predict.py` はJRAの公式ページからリアルタイムの単勝オッズをスクレイピングし、モデルの予測勝率と掛け合わせることで**期待値 (Expected Value)**を算出します。
> オッズ100倍以上の大穴馬は期待値がインフレしやすいため、クリップ処理（NaN化）が行われる仕様となっています。

## 6. 運用上の注意点まとめ

> [!WARNING]
> 長期間（数ヶ月〜半年以上）スクレイピングをサボると、差分データの量が膨大になり、取得時のエラー率上昇やIPブロックのペナルティリスク（一時的なアクセス遮断）が高まります。可能な限り**毎週または隔週での実行**を推奨します。

1. **基本は `python main.py` を週1回実行するだけ**でデータは最新に保たれます。
2. その後、必要に応じて `python train_model.py` でモデルを最新化します。
3. 最後に `predict.py` で次週の予測を行います。

## 7. コース詳細データ（umasiru）の定期メンテンナンス
JRAのコース改修や、新たな距離設定が誕生した場合は、特徴量エンジニアリングのベースとなる `all_course_features.md` を更新する必要があります。

### 実行手順
以下の2つのスクリプトを順番に実行することで、`umasiru.com` から最新のJRA全コースデータをスクレイピングし、マークダウン辞書にマージします。

```bash
# 1. umasiru.com から全コース情報をスクレイピングしてJSONを生成
python scrape_umasiru_categories.py

# 2. JSONをパースし、all_course_features.md の辞書形式で更新/追記
python append_umasiru.py
```

> [!TIP]
> 新設コースの追加や、大規模な馬場改修がない限り、このスクリプトは**半年に1回〜1年に1回程度**の実行で足ります。毎週末実行する必要はありません。
